*git config --global user.name ""
*git config --global user.email ""
 
*git init   = cria um novo repositorio local
*git clone = copia o repositorio

*git status = verificar os arquivo que foram modificados ou adicionados na pasta local.
*git add    = Adiciona na staged area, se usar o caracter “ git add . ”, adiciona todos arquivo
ou “ git add MeuArquivo.txt “ adiciona um arquivo em específico.
*git rm MeuArquivo.txt = remove arquivo
*git rm -r diretorio   = remove diretorio
*git commit -m “descricao” - o parâmetro -m é para adicionar uma descrição ao commit.
*git push origin master - envia a alteração da staged area para o repositório remoto.

* git pull  = Atualizar os arquivos no branch atual
* git fetch = Buscar as alterações, mas não aplica-las no branch atual


Desfazendo operações=======
*git checkout -- arquivo.txt    = desfaz alterações, volta o arquivo para seu estado original desde o último commit. 
*git reset meu_arquivo.txt      = tira arquivo da staging area ( . tira todos os arquivos)
*git reset --hard               = Neste caso as alterações nos arquivos também serão desfeitas com o commit


Comandos uteis ========
*ls            = lista arquivos da pasta
*cd MinhaPasta = entra na pasta
*git log       = exibir histórico (Q = sai do log)


CRIANDO NOVA BRANCH ======

*git checkout -b Nome da branch  (cria uma nova branch)
*git checkout Nome da branch (entra na branch)
*git branch    = listar branches

FAZENDO UM MERGE=======

entre na branch q vai receber os arquivos e digite:
git merge (o nome da branch que vai enviar os arquivos)
